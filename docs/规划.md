## 第一阶段：让 MaskDINO 真正处理视频

先写一个 `infer_video_maskdino.py`，完成：

```text
输入一段 MP4
→ 按 2～3 FPS 抽帧
→ 每帧运行 MaskDINO-R50
→ 绘制 mask、bbox、置信度
→ 合成为分割结果视频
→ 同时保存逐帧检测数据
```

第一版暂定：

```text
采样帧率：3 FPS
推理 batch size：1
置信度阈值：0.5
```

输出目录：

```text
output/jobs/001/
├── frames/
├── visualizations/
├── annotated_video.mp4
└── detections.csv
```

`detections.csv`：

```csv
video_id,frame_index,time_sec,instance_id,score,x1,y1,x2,y2,mask_path,crop_path
001,0,0.00,1,0.982,120,35,183,936,masks/000000_001.png,crops/000000_001.jpg
```

### 这一步的验收标准

随便选一段 20～30 秒视频，最终能够看到：

```text
原始视频
→ 连续的 MaskDINO 书脊分割视频
```

这时你就已经有了一个最基本的“视频推理结果”，视觉效果会接近 Roboflow 的示例。

## 第二阶段：加入目标书架层 ROI

因为部分视频会上下晃动、拍到相邻层，所以需要支持目标区域。

第一版不用自动检测书架层，先使用简单可靠的方式：

```text
读取视频第一张或中间帧
→ 用户框选目标书架层
→ 将 ROI 坐标保存
→ 后续所有采样帧只处理该 ROI
```

ROI 最好保存为归一化坐标：

```json
{
  "x1": 0.0,
  "y1": 0.18,
  "x2": 1.0,
  "y2": 0.82
}
```

暂时不要先开发复杂 Qt 框选控件。可以先用 OpenCV：

```python
cv2.selectROI(...)
```

等算法跑通后，再接入 Qt。

## 第三阶段：做跨帧实体去重

这是计数的核心。

同一本书可能在连续十几帧中被检测十几次，所以：

```text
检测实例数量 ≠ 实体书册数
```

你需要把重复检测合并为唯一的：

```text
book_id
```

第一版流程：

```text
相邻帧图像配准
→ 估计手机横向移动
→ 将每帧书脊位置映射到统一书架坐标
→ 根据位置重叠、尺寸和外观合并重复实例
→ 生成唯一 book_id
```

输出：

```csv
book_id,video_id,first_frame,last_frame,num_observations,best_crop_path,best_score
001_book_0001,001,0,8,9,crops/001_book_0001.jpg,0.991
001_book_0002,001,2,11,10,crops/001_book_0002.jpg,0.987
```

计数规则：

```text
一个唯一 book_id = 一册实体书
```

### 不要这样去重

不能按照 OCR 书名去重，因为书架上可能确实存在两本相同书名的书。

## 第四阶段：为每本书选择最佳 crop

一个 `book_id` 可能对应多张 crop，例如：

```text
book_001：
frame_05_crop
frame_06_crop
frame_07_crop
frame_08_crop
```

只选择最适合 OCR 的 1～3 张，评价依据：

```text
清晰度高
分割置信度高
书脊面积较大
不接触图像左右边缘
书脊完整
```

第一版可以定义一个简单分数：

```text
quality_score
= 清晰度归一化值
+ 分割置信度
+ 完整度奖励
- 边缘截断惩罚
```

最终每一本实体书只保留一个最佳 crop：

```text
output/jobs/001/books/
├── book_0001.jpg
├── book_0002.jpg
└── book_0003.jpg
```

## 第五阶段：接入 OCR

到这一步再调用百度千帆 DeepSeek-OCR。

流程：

```text
一个 book_id
→ 一张最佳 crop
→ 旋转矫正
→ OCR
→ 保存原始书名
```

先只测试 50～100 本书，不要直接处理所有视频。

输出：

```csv
book_id,crop_path,rotation,ocr_text,api_status
001_book_0001,books/book_0001.jpg,90,现代物流企业经营管埋,success
```

OCR 失败时仍保留这一本书：

```text
book_id 存在
book_title = unknown
count = 1
```

## 第六阶段：馆藏清单纠错

读取馆藏 Excel，建立标准书名词典：

```text
OCR 原始文本
→ 文本清洗
→ 模糊匹配
→ 标准馆藏书名
```

输出：

```csv
book_id,ocr_text,matched_title,match_score,status
001_book_0001,现代物流企业经营管埋,现代物流企业经营管理,0.94,matched
```

状态：

```text
matched
uncertain
unknown
unreadable
api_error
```
