# 图书盘点系统内部执行方案

成员：

* 侯舒扬
* 张梓晴

项目目标：

基于手机拍摄的图书馆书架视频，实现自动识别视频中的书名，并统计每本书的册数。最终系统需要能展示：

1. 视频输入；
2. 关键帧抽取；
3. 书脊分割结果；
4. OCR 识别结果；
5. 馆藏清单纠错结果；
6. 最终书名和数量统计表。

---

## 1. 最终技术路线

我们最终采用：

```text
实例分割模型微调（书脊分割）
+ 百度千帆 DeepSeek-OCR 接口调用（书名识别）
+ 馆藏清单候选词典纠错（可选增强）
+ 视频多帧融合计数
+ 可视化软件展示
```

整体流程：

```text
手机视频
↓
关键帧抽取
↓
微调后实例分割模型输出书脊 mask + bbox
↓
裁剪单本书脊 crop
↓
百度千帆 DeepSeek-OCR 接口识别书名
↓
馆藏清单模糊匹配纠错
↓
多帧融合去重计数
↓
输出书名 + 册数
```

## 3. 最终方案定位

最终口径：

```text
我们自行采集书架视频，构建书脊轮廓标注数据集，并在该数据集上对实例分割模型进行任务适配微调，
使其能够稳定分割细长、密集、倾斜的书脊目标。
书名识别部分通过百度千帆 AI 开发者中心的 DeepSeek-OCR 接口识别裁剪后的单本书脊图像，
并在需要时结合馆藏清单做候选书名模糊匹配纠错。
最后通过视频多帧融合策略解决重复计数和漏计问题。
```

## 5. OCR 部分方案

### 5.1 OCR 方案

OCR 通过百度千帆 AI 开发者中心的 DeepSeek-OCR 接口完成，不做微调。后处理可以再叠加馆藏清单模糊匹配纠错来提升准确率。

接口落地时统一按下面这组参数实现：

```text
请求地址：https://qianfan.baidubce.com/v2/chat/completions
model：deepseek-ocr
鉴权：Authorization: Bearer <API Key>
限制：单轮、单张图像识别
```

### 5.2 OCR 不直接识别整图

不直接把整张书架图输入 OCR。

原因：

```text
1. 书脊文字很密；
2. 竖排书名多；
3. 多本书文字会互相干扰；
4. 反光和倾斜会影响识别；
5. 整图 OCR 容易把多个书名混在一起。
```

正确流程：

```text
分割模型输出单本书脊 mask
↓
裁剪单本书脊 crop
↓
旋转矫正
↓
调用百度千帆 DeepSeek-OCR 接口识别书名
```

### 5.3 OCR 输出格式

要求模型只输出书名，不要解释。

错误输出：

```text
这本书可能是《现代物流企业经营管理》
```

正确输出：

```text
现代物流企业经营管理
```

Prompt 示例：

```text
请识别这张书脊图片中的书名，只输出书名，不要解释。
```

### 5.4 OCR 指标

报告里可以放：

```text
OCR 原始 Top-1 准确率
OCR + 旋转矫正准确率
OCR + 馆藏清单纠错准确率
字符错误率 CER
Top-3 命中率
```

## 6. 馆藏清单纠错方案

### 6.1 词典来源

使用：

```text
泰达西区库-馆藏清单-按册20260522-005.xlsx
```

### 6.2 纠错流程

```text
百度千帆 DeepSeek-OCR 输出
↓
文本清洗
↓
如有需要，再与馆藏清单候选书名匹配
↓
输出标准馆藏书名
```

匹配方法：

```text
编辑距离
最长公共子序列
关键词包含度
拼音相似度
```

### 6.3 匹配规则

```text
如果 top1 相似度 ≥ 0.75：
    采用 top1 馆藏书名

如果 0.55 ≤ top1 相似度 < 0.75：
    标记 uncertain，交给多帧投票或人工复核

如果 top1 相似度 < 0.55：
    标记 unknown
```

### 6.4 报告口径

可以写：

```text
由于书脊 OCR 容易出现错字、漏字和竖排识别错误，
本项目利用馆藏清单构建候选书名词典，将自由文本识别结果映射到标准馆藏书名，
从而提高书名识别稳定性。
```

## 7. 视频计数方案

### 7.1 为什么不能逐帧相加

如果直接把每一帧识别到的书都相加，会出现：

```text
同一本书在多个关键帧中重复计数
```

所以要做视频级融合。

### 7.2 简单稳定计数

每个视频抽 2～3 张关键帧：

```text
每帧分割
↓
每帧 OCR
↓
每帧得到书名列表
↓
同一视频内按书名聚合
↓
输出最终 count
```

计数规则：

```text
固定拍摄视频：
    对同一书名取多帧中的最大稳定数量或中位数

横向扫书架视频：
    使用书名一致性 + 位置连续性 + mask IoU 做合并
```

简单例子：

```text
frame_1：物流配送中心管理 2 本
frame_2：物流配送中心管理 3 本
frame_3：物流配送中心管理 3 本

最终输出：物流配送中心管理 3 本
```

### 7.3 视频级真值标注

必须建立：

```text
annotations/video_gt.csv
```

格式：

```csv
video_id,book_title,count,notes
001,现代物流企业经营管理,1,
001,物流配送中心管理,2,
002,数字图像处理,1,
```

这个表必须做，因为最后需要证明系统结果是否准确。

## 13. 答辩口径

### 老师问：你们是不是只调用大模型？

回答：

```text
不完全是。分割部分我们自行采集视频、构建标注数据集，并在该数据集上对实例分割模型进行任务适配微调。
OCR 部分确实通过百度千帆 AI 开发者中心的 DeepSeek-OCR 接口完成，但我们结合了馆藏清单进行候选书名模糊匹配纠错，
并通过视频多帧融合策略解决重复计数问题。
```

### 老师问：你们的数据标注在哪里？

回答：

```text
我们有两层标注：
第一，书脊轮廓 polygon 标注，共 229 张，用于分割模型训练；
第二，视频级书名和册数真值 video_gt.csv，用于最终计数评价。
```

### 老师问：为什么不把所有帧都手工标？

回答：

```text
视频帧数量非常大，逐帧手工标注成本过高。
因此我们采用人工精标少量关键帧、训练初版模型、模型辅助预标注、人工复核修正的方式，
既保证标注质量，也提高标注效率。
```

### 老师问：怎么保证计数准确？

回答：

```text
我们没有简单累加每帧结果，而是在视频级别进行多帧融合。
同一书名在多帧中重复出现时，会通过多帧投票、最大稳定计数和位置一致性进行合并，
从而减少重复计数和漏计。
```

### 老师问：OCR 为什么不微调？

回答：

```text
考虑到时间成本和百度千帆 DeepSeek-OCR 接口本身已经有较强的通用识别能力，
我们选择直接调用接口，并通过后处理（旋转矫正 + 可选的馆藏清单纠错）来提升准确率。
实验结果表明这种方案在书脊识别任务上效果良好。
```
