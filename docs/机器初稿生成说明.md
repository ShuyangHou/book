# 机器初稿生成说明

这一步的目标不是直接得到最终真值，而是先让机器给出一版可以人工修改的初稿。

当前最终采用的接入方式是：

- 直接调用百度千帆 AI 开发者中心的 `DeepSeek-OCR`
- 请求地址固定为 `https://qianfan.baidubce.com/v2/chat/completions`
- `model` 固定写 `deepseek-ocr`
- 用 `Authorization: Bearer <API Key>` 鉴权

这一点在报告和答辩口径里要写清楚。

整体流程是：

```text
书脊 crop
-> OCR 结果 CSV
-> 书名纠错 / 汇总
-> pred_inventory.csv
-> 人工修正成 video_gt.csv
```

## 1. 你们现在需要准备什么

如果你们还没有 OCR 结果 CSV，可以先直接用项目里的脚本：

[ocr_qianfan.py](C:\Users\15075\Desktop\课程作业\项目制实践-AI\_tools\ocr_qianfan.py:1)

它支持：

1. 本地图片转 Base64 后调用千帆的 `deepseek-ocr`
2. 按 `0 -> 90 -> 270` 顺序按需重试
3. 按图片哈希缓存，避免重复扣费
4. 如果你们后面还想加词典纠错，也可以再额外接一个馆藏词典文件

最小运行方式：

```bash
set QIANFAN_API_KEY=YOUR_API_KEY
python _tools/ocr_qianfan.py ^
  --input-dir crops ^
  --glob *.jpg ^
  --output-csv output/ocr_results/ocr_results.csv ^
  --attempts-csv output/ocr_results/ocr_attempts.csv ^
  --cache-jsonl output/ocr_results/ocr_cache.jsonl
```

Linux 写法：

```bash
export QIANFAN_API_KEY=YOUR_API_KEY
python _tools/ocr_qianfan.py \
  --input-dir crops \
  --glob '*.jpg' \
  --output-csv output/ocr_results/ocr_results.csv \
  --attempts-csv output/ocr_results/ocr_attempts.csv \
  --cache-jsonl output/ocr_results/ocr_cache.jsonl
```

如果你们不是直接让脚本调用 OCR，而是先手工整理结果表，最少要有两列：

```csv
image_path,ocr_text
```

推荐完整些：

```csv
image_path,ocr_text,confidence,matched_title
```

说明：

1. `image_path`
   可以是 `001_03s_crop_0001.jpg` 这种文件名，脚本会自动解析出 `video_id` 和 `frame_id`。
2. `ocr_text`
   是 OCR 接口原始识别结果。
3. `confidence`
   可选。
4. `matched_title`
   可选。如果你们已经自己做过馆藏匹配，就把标准书名填进来；脚本会优先用这一列。

## 2. 现成脚本

我已经加了脚本：

[ocr_qianfan.py](C:\Users\15075\Desktop\课程作业\项目制实践-AI\_tools\ocr_qianfan.py:1)

和

[build_pred_inventory.py](C:\Users\15075\Desktop\课程作业\项目制实践-AI\_tools\build_pred_inventory.py:1)

它们配合起来完成两步：

1. `ocr_qianfan.py`
   生成 `ocr_results.csv`
2. `build_pred_inventory.py`
   把 `ocr_results.csv` 汇总成 `pred_inventory.csv`

第二步会做三件事：

1. 把每个 crop 的 OCR 结果清洗并尝试匹配到馆藏书名。
2. 统计每一帧里每本书出现了几次。
3. 按视频聚合，生成一版草稿 `pred_inventory.csv`。

## 3. 最简单的用法

假设你们已经有一个 OCR 结果文件：

```text
你们自己整理的 OCR 结果 CSV
```

直接运行：

```bash
python _tools/build_pred_inventory.py ^
  --ocr-csv 你们自己的OCR结果.csv ^
  --per-crop-out output/ocr_results/ocr_matched.csv ^
  --frame-counts-out output/final_inventory/frame_counts.csv ^
  --inventory-out output/final_inventory/pred_inventory.csv ^
  --aggregate max ^
  --keep-unknown
```

Linux 写法：

```bash
python _tools/build_pred_inventory.py \
  --ocr-csv 你们自己的OCR结果.csv \
  --per-crop-out output/ocr_results/ocr_matched.csv \
  --frame-counts-out output/final_inventory/frame_counts.csv \
  --inventory-out output/final_inventory/pred_inventory.csv \
  --aggregate max \
  --keep-unknown
```

## 4. 会生成哪些文件

### 4.1 `output/ocr_results/ocr_matched.csv`

每个 crop 的详细匹配结果，适合排查 OCR：

- 原始文本
- 清洗后文本
- 匹配到的标准书名
- 匹配分数
- 匹配状态

### 4.2 `output/final_inventory/frame_counts.csv`

按帧统计的结果，适合看某一帧里某本书被数了几次。

### 4.3 `output/final_inventory/pred_inventory.csv`

这是最重要的机器初稿，格式和 `video_gt.csv` 一样，可以直接人工修改。

## 5. `--aggregate` 该怎么选

脚本支持三种视频级聚合：

1. `max`
   取同一本书在多帧中的最大计数
2. `median`
   取中位数
3. `mode`
   取众数

目前最推荐先用：

```text
--aggregate max
```

因为你们的视频里更怕漏计，先用 `max` 更容易得到一版完整初稿，后面再人工删掉重复项。

## 6. 你们应该怎么实际操作

### 方案 A：最省事

1. 先让 OCR 接口把所有 crop 跑一遍。
2. 把结果整理成一个 OCR CSV。
3. 跑 `build_pred_inventory.py`。
4. 打开生成的 `pred_inventory.csv`。
5. 在它的基础上人工修正，拷贝到 `video_gt.csv`。

### 方案 B：更稳一点

1. OCR 跑一遍。
2. 先检查 `ocr_matched.csv` 里明显错得离谱的项。
3. 再看 `pred_inventory.csv` 做视频级修正。

## 7. 如果你们已经自己做了馆藏纠错

那就直接在 OCR CSV 里填 `matched_title` 列。

脚本会优先用 `matched_title`，不会再重复 fuzzy match。

这很适合你们后面把 DeepSeek-OCR 和馆藏词典匹配脚本接起来。

## 8. 一句话理解这一步

这个脚本生成的 `pred_inventory.csv` 不是最终真值，而是**机器先写的初稿**。你们真正的省时间方式就是：

```text
先让机器写 70% 到 80%
再人工改最后那 20% 到 30%
```

这样会比从空白 `video_gt.csv` 开始填快很多。
